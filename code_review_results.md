## Описание алгоритма

Алгоритм проверяет, заполнил ли пользователь поля формы:
1. Проверяет поле "Имя" — не пустое ли
2. Проверяет поле "Телефон" — не пустое ли
3. Проверяет поле "Email" — не пустое ли
4. Проверяет длину телефона (не менее 10 цифр)

Если какая-то проверка не пройдена — выводится сообщение об ошибке.
Если все проверки пройдены — данные принимаются.

## Что нужно улучшить (замечания)

### 1. Проверка формата email
- **Проблема:** Алгоритм проверяет только наличие email, но не проверяет, правильно ли он написан
- **Пример ошибки:** Пользователь введёт "абвгд" — алгоритм пропустит, но это не email
- **Как исправить:** Добавить проверку, что в email есть символ @ и точка после него

### 2. Проверка формата телефона
- **Проблема:** Алгоритм проверяет только длину, но не проверяет, из чего состоит телефон
- **Пример ошибки:** Пользователь введёт буквы вместо цифр
- **Как исправить:** Добавить проверку, что телефон состоит только из цифр

### 3. Обработка пробелов
- **Проблема:** Если пользователь поставит пробелы, алгоритм может не заметить
- **Пример ошибки:** Пользователь введёт "   " — алгоритм решит, что поле заполнено
- **Как исправить:** Перед проверкой удалять лишние пробелы

### 4. Показ ошибок
- **Проблема:** Ошибки показываются по одной, это неудобно
- **Пример:** Пользователь не заполнил три поля — ему покажут ошибку про первое, потом про второе и т.д.
- **Как исправить:** Собирать все ошибки в список и показывать их все сразу

## Предложения по улучшению

1. Добавить проверку email (должен содержать @ и точку)
2. Добавить проверку телефона (только цифры)
3. Перед проверкой удалять пробелы из начала и конца строк
4. Собирать все ошибки в один список и показывать пользователю сразу
5. Выделять красным поля, в которых допущена ошибка

## Заключение

Алгоритм выполняет свою основную задачу, но имеет несколько недостатков. После исправления замечаний (добавление проверки форматов, обработка пробелов, групповой показ ошибок) алгоритм станет более надёжным и удобным для пользователя.
