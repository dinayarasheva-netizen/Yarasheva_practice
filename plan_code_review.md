# План проверки алгоритма (Code Review)

## Алгоритм для проверки
Проверка данных формы обратной связи (имя, телефон, email)

## Критерии оценки

### 1. Правильность работы (0-5)
- Алгоритм решает поставленную задачу?
- Все ли случаи обработаны?

### 2. Понятность (0-5)
- Легко ли понять логику алгоритма?
- Логичная ли последовательность шагов?

### 3. Полнота проверок (0-5)
- Все ли поля проверяются?
- Проверяется ли правильность формата данных?

### 4. Удобство для пользователя (0-5)
- Понятные ли сообщения об ошибках?
- Удобно ли пользователю исправлять ошибки?

### 5. Надёжность (0-5)
- Есть ли защита от неверного ввода?
- Корректно ли обрабатываются пробелы?

### 6. Простота изменений (0-5)
- Легко ли добавить новое поле?
- Легко ли изменить правила проверки?

## Шкала оценки
- 5 — отлично, замечаний нет
- 4 — хорошо, есть мелкие недочёты
- 3 — удовлетворительно, есть серьёзные недочёты
- 2 — плохо, много ошибок
- 1 — очень плохо, алгоритм не работает

## Итоговая формула
(Сумма баллов) / 30 × 100% = качество алгоритма в процентах
